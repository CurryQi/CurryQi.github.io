IPv6报文头部

IPv6报头是IPv4报头的精简版，去除了非必须的和很少使用的字段，并添加了一个字段用于更好的支持实时传输。

  • 版本：4 bits，定义IP协议的版本，设置为6，值在协议层不曾使用，识别过程在链路层通过协议标识字段已经完成；例如 Ethernet II 中使用 16 位以太类型字段标识以太网负载：IPv4 是 0x0800，IPv6 是 0x86DD。
  • 传输类型：8 bits，定义IPv6包的类型或优先级，功能近似IPv4的区分服务字段，前6 bits用于DSCP，后2 bits用于ECN。
  • 流标签：20 bits，标识源和目标之间一系列包中的一个，需要中间路由特殊处理，用于有序发送，比如实时数据（语音或视频）的发送，流标签字段设置 0 时表示默认的路由处理方法。中间路由可以使用包的源地址、目的地址和流标签以辨别特定的流。源和目的之间能够有多个流，用非零流标签辨别。‘
  • 有效长度负载：16 bits，定义IPv6有效负载长度，包括了扩展报头和上层PDU，能够表示上限为 65535 字节的IPv6有效负载，对超过上限的有效负载，本字段设置为0，并动用逐跳选项报头中的超大有效负载选项。
  • 下一报头：8 bits，定义首个扩展报头（假设存在）或上层PDU协议的类型。当为上层PDU协议类型时，本字段和IPv4的协议字段中使用相同的值。
  • 跳数：8 bits，定义IPv6包能经过的最大链路数，超过此数的包会被丢弃，类似于IPv4的TTL字段，若某个包在路由器中跳数为0，路由器会发送ICMPv6超时报文到源主机，并丢弃此包。
  • 源地址：128 bits，定义起始主机的IPv6地址
  • 目的地址：128 bits，定义目的节点的IPv6地址，多数情况下目的地址字段设置为最终目的地址，但如果有路由扩展报头，目的地址字段会设置为下一中间目的的地址。

IPv6下一报头

值（十进制） 报头
0 逐跳选项报头
6 TCP
17 UDP
41 已封装的IPv6报头
43 路由报头
44 片段报头
50 已封装的安全有效负载报头
51 验证报头
58 ICMPv6
59 没有下一报头
60 目标选项报头
  • IPv6设计者们希望能够优化中间路由器上处理IPv6数据包的过程，唯一必须在中间每一个路由器中处理的扩展报头就是逐跳选项报头。
  • 为了优化测试逐跳选项报头，是否存在这一个过程，将该值设置为0，在路由器硬件过程中，判断0比判断59更容易。

IPv4 & IPv6路由转发

IPv4路由转发关键步骤

  1. 校验报头校验和。路由器执行自己的校验和计算并将结果与IPv4报头中的校验和进行比较，这个步骤在RFC 1812中是必须的，但是现代高速路由器通常跳过。
  2. 校验版本字段。虽然在RFC 791 或 RFC 1812中是非必须的，但是执行该步骤能够节省网络带宽，因为包含无效版本的数据包通过IPv4互联网传播，最终只会被目标节点丢弃。
  3. 递减TTL字段的值。如果新的值小于1，就发送ICMPv4超时（TTL超时）报文给源地址，并丢弃该数据包。
  4. 检查是否存在IPv4报头选项。如果存在，就处理选项。
  5. 使用目的地址的值和本地路由表的内容决定转发接口、下一跳IPv4地址。如果路由没有找到，就发送ICMPv4目标不可达（主机不可达）报文给源地址，并丢弃数据包，
  6. 检查转发接口IPv4 MTU，若转发接口IPv4 MTU值小于总长度字段的值，并且“不分片字段（DF，dont‘t Fragment）”为0，执行IPv4分片操作；若转发接口IPv4 MTU小于总长度字段的值，并且DF=1，则发送ICMPv4目标不可达（需要分片和DF置为）报文给源地址，并丢弃数据包。
  7. 重新计算新的报头校验和，并更新到报头的校验和字段。
  8. 使用合适的转发接口转发该数据包。

IPv6路由转发关键步骤

  1. 校验版本字段，虽然该步骤在RFC 2460中是非必须的，但执行该步骤能节省网络带宽，与IPv4相同步骤同理。
  2. 递减跳数字段的值，如果新的值小于1，就发送ICMPv6超时（跳数限制超时）报文给源地址，并丢弃该数据包。
  3. 检查下一报头字段是否为0，若为0，则处理逐跳选项报头。
  4. 使用目的地址的值和本地路由表的内容决定转发接口、下一条IPv6地址。如果路由没有找到，就发送ICMPv6目标不可达（没有到达目标的路由）报文给源地址，并丢弃数据包，
  5. 检查转发接口IPv6 MTU，若转发接口链路MTU小于40加上有效负载字段的总和，则向源地址发送ICMPv6数据包过长报文，并丢弃数据包。
  6. 使用合适的转发接口转发数据包。

小结对比

IPv6数据包转发过程比IPv4数据包要简单得多，因为减少了两次校验的环节，不需要执行拆分操作，不需要处理本就不该由路由器处理的选项。

IPv6扩展报头

IPv4报头包含了所有的选项，每个中间路由器都必须检查它们是否存在，如果存在，则需要进行处理，这降低了IPv4数据包转发的效率。

IPv6中，发送和转发选项被移至扩展报头中，中间路由器唯一必须处理的扩展报头就是逐跳选项扩展报头，这加快了IPv6数据包的转发效率。

扩展报头的顺序：

  1. 逐跳选项报头
  2. 目标选项报头（当路由报头存在时，对于中间目标而言）
  3. 路由报头
  4. 片段报头
  5. 身份验证报头
  6. 封装安全有效负载报头
  7. 目标选项报头（最终目标）

逐跳选项报头

作用：为发送到目标主机途中的每一次跳转指定转发参数。

识别方法：下一跳报头值为0。

结构：

  • 下一报头
  • 报头扩展长度：本字段为逐跳选项报头中8字节块的块数，不包含第一个8字节。
    • 若逐跳选项报头刚好为8字节，本字段值为0
    • 若逐条选项报头不为8字节，使用填充选项进行填充为8的整数倍。
  • 选项（一个或多个，采用TCP/IP协议常用的类型-长度-值，即TLV编码格式）
    • 选项类型：既要标识选项本身，又要标识节点对其的处理方法
    • 选项长度：标识选项中的字节数，不包括选项类型和选项长度字段
    • 选项数据：与选项相关的数据

选项可能由对其要求，以保证选项中特定字段正好位于边界上，对其要求用表达式 xn + y 标识，含义为选项必须从 x 字节的整数倍，加上 y 字节的字节边界起始。为了满足对其要求，多个选项时，通常使用填充字段置于某个选项之间或每个选项之间。

选项类型字段

值（二进制） 动作
00 跳过此选项
01 自行丢弃此包
10 丢弃此包，若IPv6报头中目的地址为单播或多播，发送ICMPv6参数问题信息到源地址
11 丢弃此包，若IPv6报头中目的地址不是多播地址，发送ICMPv6参数问题信息到源地址

Pad 1选项

占1字节，用于插入单个填充字节，使逐跳选项或目标选项报头位于8字节边界并满足选项对其要求。本选项自身没有对其要求。

Pad N选项

由选项类型字段和长度字段组成。

  • 选项类型字段：设置为1时，可以跳过无法识别的选项，并不允许在传输过程中改变选项。
  • 长度字段：填充字节数

超大有效负载选项

RFC 2675 定义了本选项，用于表示超过 65535 字节的有效负载的大小。本选项需要满足 4n + 2 对齐要求。

若存在本选项，则IPv6报头中的有效负载长度字段 = 0，本选项占32位，最大能够标识长度位42 9496 7295字节的有效负载，IPv6包中的有效负载超过 65535 字节时称位超大包，当选项类型字段设置位 194 时，执行：丢弃包、发送ICMPv6参数问题信息到源地址，不允许在传输中改变选项。

Windows Vista、Windows Server 2008 和Window XP-SP 2版系统下，IPv6协议支持进入超大包，但是TCP、UDP不支持发送和接收超大包。

路由器告警选项

RFC 2711 定义路由器告警选项（选项类型5），用于告知路由器：包中内容需要额外处理。本选项有 2n + 0 对齐要求。主要用于多播侦听发现（MLD）和资源存储协议（RSVP）。当选项类型位5时，可以跳过无法识别的选项，并且不允许在传输过程中改变选项。

目标选项报头

用于为中间目标或最终目标指定发送包的参数，此报头由前一报头的下一报头字段值60表示。结构与逐跳选项报头类似。

  • 若存在路由报头，且目标选项报头出现在路由报头前，则目标选项报头表示每个中间目标的发送或处理选项
  • 目标选项报头出现在路由报头后，，则表示最终目标的发送或处理选项。

本地地址选项

RFC 3775 定义本地地址目标选项（选项类型201），此选项用于表示移动节点的本地地址。当移动节点连到本地链路时，分配给它本地地址。通过本地地址，不管移动节点在IPv6网络哪里，它总是可达的。本地地址选项有 8n + 6 对齐要求。

结构：

  • 选项类型：设置为201，十六进制0xC9，二进制0b11001001，一旦选项被无法识别，并且目的地址不是多播地址，则数据包将被丢弃，同时发送ICMPv6参数问题报文， 在传输中也不允许修改选项。
  • 选项长度：16 bits，以字节为单位表示选项长度，并不包括选项类型和选项长度字段，作用时存储IPv6地址的本地地址地段字段。
  • 本地地址：128 bits，表示移动节点的本地地址。

选项类型总结

选项类型 选项及其用途 对其要求
0 Pad 1 选项，逐跳和目标选项报头 无
1 Pad N 选项，逐跳和目标选项报头 无
194（0xC2） 超大有效负载选项：逐跳选项报头 4n + 2
5 路由器告警选项：逐条选项报头 2n + 0
201（0xC9） 本地地址选项：目标选项报头 8n + 6

路由报头

IPv4定义了严格的源路由，其中每个中间路由器只能跳一次，同时也定义了自由源路由，其中每个中间路由器能够跳一次或多次。IPv6源路由器可以使用路由报头来指定源路由，即中间目标的列表，而数据包正是在这个通路上传输并到达最终的目标。路由报头通过前一报头的下一报头设置为43来标识。

路由报头结构：

  • 下一个报头
  • 报头扩展长度
  • 路由类型
  • 段剩余
  • 特定路由类型的数据

路由类型0报头的结构：

  • 下一个报头
  • 报头扩展报头
  • 路由类型 = 0
  • 段剩余：定义为指定路由类型的数据中总共的地址数目
  • 保留字段
  • 地址1
  • ……
  • 地址N

对于路由类型0，指定了路由类型的数据包括32位保留字段以及一个包含最终目的地址的中间目标地址列表，当数据包最初被发送时，目的地址设置位首个中间目标，而指定了路由类型的数据就是其他的中间目标列表以及最终目标。

  • 当数据包到达中间目标时，会处理路由报头并执行如下操作
    1. 交换当前目标地址和列表中第（N - 段剩余 + 1）位置的地址，其中N是路由报头中的总地址数
    2. 递减段剩余字段
    3. 转发数据包
  • 当数据包达到最终目标时，段剩余字段以及被设置为0，而到达最终目标之前的链路上以及访问过的中间地址列表会记录在路由报头中。

Windows Vista中IPv6会接收并处理包头中的路由类型为0的进入数据包。

但是基于安全性考虑，IETF不推荐支持路由类型0的数据包，Windows Server 2008和Windows Vista Server Pack 1中IPv6会丢弃路由类型0的进入数据包。

移动IPv6使用类型2路由报头。
