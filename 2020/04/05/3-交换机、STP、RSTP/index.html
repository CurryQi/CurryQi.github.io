3 交换机、STP、RSTP
发表于 2020-04-05| 更新于 2020-04-05|路由器初级
阅读量:|评论数:

1 交换原理

1.1 交换机概括

交换机工作在数据链路层，转发数据帧，隔离冲突域

1.2 转发行为

交换机一共有3中转发行为：

  • 泛洪（Flooding）：把某一端口的数据帧通过所有其他端口转发出去
  • 转发（Forwarding）：把某一端口的数据帧通过其他任一端口转发出去
  • 丢弃（Discarding）：把某一端口的数据帧直接丢弃

1.3 工作原理

  1. 单播帧进入交换机，检查MAC地址表，查询是否存在单播帧目的MAC地址
    • 若没有找到，执行泛洪
    • 若找到了，比较MAC地址对应的端口是否为单播帧进入交换机的端口
      • 若不是，转发
      • 若是，丢弃
  2. 广播帧进入交换机，不检查MAC地址表，直接执行泛洪
  3. 组播帧

1.4 工作流程

  1. 初始状态下，交换机MAC地址表为空

  2. 学习MAC地址与端口映射

    • 收到数据帧时，将数据帧源MAC地址和输入端口的映射关系记录在MAC地址表中

    • 发送数据帧时，首先发送ARP请求，利用ARP回复报文获取输出端口，添加目的MAC地址与输出端口的映射关系到MAC地址表中

    • 缺省情况下，一般交换机的MAC地址表项老化时间为300秒，若在老化时间内继续发送或收到回复数据，则刷新对应端口映射的老化时间

  3. 数据帧转发

  4. 目标主机回复

1.5 交换机配置

1.6 Q&A

Q：一台主机交换机从一个端口移动到另外一个端口，交换机的MAC地址表会发生什么变化

A：首先交换机检测到旧的物理链路失效，从MAC地址表中移除对应的MAC地址与端口映射关系；

​ 当主机重新连接到另外一个端口时，建立新的物理连接；

​ 当主机发送数据或者接收数据时，生成新的MAC地址表与端口映射关系，并记录到MAC地址表中。

2 STP原理及配置

2.1 环路引发的问题

  • 为了提高网络可靠性，避免单点故障时业务中断，交换网络中常常需要使用冗余链路实现备份
  • 冗余链路给交换网络带来环路风险
    • 广播风暴：由于交换机的泛洪规则，当交换网络存在环路时，广播帧会被无限转发，形成风暴，网络会充斥着重复的广播帧
    • MAC地址表震荡：交换机是根据收到的数据帧的源地址和接收端口，生成MAC地址表项；数据帧接收端口可能由于环路原因，重新被另外的端口接收，之前的MAC地址表会被删除，生成新的表项，这个过程不断重复，从而导致MAC地址表震荡

2.2 STP的功能

  • 消除环路：阻塞冗余链路端口
  • 链路备份：当活动路径发生故障，激活备份链路，回复网络连通性

2.3 STP工作流程

STP通过构造一棵无环树状网络来消除交换网络中的环路

2.3.1 STP角色介绍

  • BPDU：根桥选举时交换机之间发送的交互信息，包含了参加生成树计算的各种参数信息
  • 根桥：又名根交换机，位于整颗逻辑树的顶部
  • 根端口：非根交换机去往根桥的最优端口
  • 指定端口：交换机所连网段转发配置BPDU的端口
  • 预备端口：若一个端口既不是根端口，也不是指定端口，则为预备端口

2.3.2 根桥选举

  • 每个交换机的BID（桥ID），由16位桥优先级和48位MAC地址表构成
  • 优先级取值范围位0~65535，默认为32768
  • BID越小优先级越高
    • 所有交换机开始均认为自己为根桥，且自身端口均为指定端口；
    • 收到BPDU后，比较BPDU中的BID和自身BID
      • 若自身BID优先级高，继续通告自身配置的BPDU给邻居交换机
      • 若BPDU中的BID优先级高，则修改自己的BPDU保温的BID字段，宣告新的根桥

2.3.3 根端口选举

非根交换机选举根端口时，分别依据根路径开销、对端BID、对端PID、本段PID进行选举

  1. 根路径开销
    • 端口开销：交换机每个端口都有端口开销参数，默认情况下该参数与端口带宽有关，带宽越高开销越小
    • 路径开销：非根交换机到达根交换机路径上所有接收BPDU端口的端口开销总和
    • 根路径开销：非根交换机到达根交换机最短的路径开销，根桥的根路径开销是0
  2. 对端BID
    • 对端口的BID，越小优先级越高
  3. 对端PID
    • 交换机每个端口都有一个端口ID，即PID，由端口优先级和端口号组成
    • 端口优先级取值范围为0~240，步长为16，即取值必须为16的整数倍
    • 缺省情况下，默认PID为128
  4. 本段PID
    • 本端口的PID，越小优先级越高

2.3.4 指定端口选举

在每个网段上指定端口选举时，根据根路径开销、BID、PID进行选举

  • 指定端口：非根交换机在网段上抑制其他端口发送BPDU报文的端口
  • 每个网段都有一个指定端口
  • 根桥上的每一个端口都是指定端口

2.3.5 阻塞预备端口

当根桥选举、根端口选举、指定端口选举均完成时，阻塞预备端口。

2.4 端口转换与BPDU

2.4.1 端口转换

STP端口状态一共有五个：

  1. Disabled：禁用状态，
  2. Blocking：阻塞状态，端口仅仅接收并处理BPDU
  3. Listening：侦听状态，端口仅仅转发BPDU
  4. Learning：学习状态，端口根据收到的用户流量构建MAC地址表
  5. Forwarding：转发状态，端口既可以转发用户流量，也可以转发BPDU报文，且只有根端口和指定端口能够进入该状态

2.4.2 BPDU

  1. 为了计算生成树，交换机之间需要交换相关信息和参数，这些信息和参数被封装到BPDU里

  2. BPDU分两种类型：

    • 配置 BPDU：用于根桥选举和生成树的计算
    • TCN BPDU：下游交换机感知拓扑发生变化时向上游发送通知

  3. 配置 BPDU 包含足够的信息以保证完成生成树计算
