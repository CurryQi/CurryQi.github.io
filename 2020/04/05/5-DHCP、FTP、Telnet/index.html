1 DHCP

主要用于为大量的网络终端，动态分配IP地址，并进行集中管理

1.1 报文类型：

报文类型 含义
DHCP Discover 客户端寻找DHCP服务器
DHCP Offer 携带各种配置信息，用于DHCP服务器响应客户端的寻找报文
DHCP Request 客户端请求配置确认，或者续借租期
DHCP ACK 服务器对客户端请求报文确认响应
DHCP NAK 服务器对客户端请求报文拒绝响应
DHCP Release 客户端释放地址时通知服务器

1.2 DHCP 工作原理

  1. Client 初次接入网络，发送 Discover(广播)
  2. Server 收到 Discover(广播) 报文，发送 Offer(广播/单播) 提供一个未分配的IP地址等配置信息
  3. Client 收到 Offer(广播/单播) 报文，选择第一个到达的，发送带有IP地址的 Request(广播)，确认配置信息
  4. Server 收到 Request，回复 ACK 提供IP地址和配置信息进行确认
  5. Client 收到确认报文，发送免费 ARP 报文，用于检查网络中是否有其他主机使用分配的地址
  6. 若指定时间内没有收到ARP响应，Client 会使用这个IP地址
  7. 若有主机占用该IP地址，Client 发送拒绝报文，通知 Server 该IP已被占用，并重新申请一个IP地址
  8. Client 获取的IP地址租期快到期时，发送 Request 申请延长IP使用时间

1.3 地址池

地址池 功能 优先级
接口地址池 为连接到同一网段的主机或终端分配IP地址 高
全局地址池 为所有连接到DHCP服务器的终端分配IP地址 低

1.4 DCHP租期更新与重绑定

  • 申请到IP地址后，Client 保存三个定时器，分别用来控制租期更新、租期重绑定、租期失效
  • 租期更新
    1. Server 为 Client 分配IP地址时，指定3个定时器的值
    2. 若未指定，Client 使用缺省值(默认)，缺省租期为1天
    3. IP租约期限到达50%时，Client 会重新请求更新IP地址的租约
  • 租期重绑定
    1. Client 发送 Request(单播) 报文续租时，若 Client 收 Server 的应答报文超时，Client 认为原 Server 不可用
    2. 重新发送 Request(广播)，网络中任意一台 Server 均可应答 ACK 或 NAK 报文
    3. 收到 ACK 后，Client 重新进入绑定状态，复位租期更新定时器、重绑定定时器
    4. 收到 NAK 后，Client 重新进入初始化状态，Client 立即停止现有 IP 地址，重新申请IP地址
    5. IP租约期限达到87.5%时，且未收到 Server 的更新响应，申请重绑定
  • 租期失效
    1. IP地址租约到期，且仍未收到 Server 的响应，Client 立即停止使用此IP地址
    2. 若 Client 不再使用分配的IP地址，可以主动向 Server 发送 Release 报文，释放该地址
    3. 租期失效在租约到期后执行，而后 Client 进入初始化状态，重新发送 Discover 报文，申请IP地址

1.5 DHCP配置

  • 接口地址池
  • 全局地址池

1.6 DHCP配置命令

命令 功能
dhcp enable 使能DHCP
dhcp select interface 关联当前接口和接口地址池，为连接到接口的主机提供配置信息
dhcp server dns-list 指定接口地址池下的DNS服务器地址
dhcp server excluded-ip-address 配置接口地址池中不参与自动分配的地址
dhcp server lease 配置IP地址的租用有效期，缺省状态下为1天
display ip pool 查看地址池信息
ip pool 创建全局地址池
network 配置全局地址池下可分配的网段地址
gateway-list 配置DHCP服务器全局地址池的出口网管地址
lease 配置DHCP全局地址池下的地址租期，缺省状态下为1天
dhcp select global 使能当前接口的DHCP服务器功能

2 FTP

用于文件传输的协议，使用FTP实现远程文件传输的同时，可以保证数据传输的可靠性和高效性

2.1 FTP应用场景

  • 部署一台FTP服务器，终端设置为FTP客户端，使用FTP进行VRP和配置文件的更新
  • 将任意网络设备配置为FTP服务器，将该设备的日志文件保存到终端上，方便查看

2.2 FTP传输原理

  1. 使用两个TCP连接，一个连接为 Client-Server 控制接口，另一位连接为 Client-Server 数据接口
  2. Server 开启 21 号端口，等待 Client 发送连接请求；Client随机开启端口，发送请求，用于传输控制命令
  3. Server 开启 20 号端口，与客户建立数据连接，一般数据连接由 Server 主动建立、主动断开

2.3 FTP传输模式

  • ASCII模式：用于传输文本，发送端的字符转换为ASCII码后进行传输，接收端将ASCII码转换为字符
  • 二进制模式：用于传输图片文件和程序文件，文件传输时无需转换格式

2.4 FTP配置

  • AAA认证配置FTP：为每一个用户分配登录账号和访问权限
  • 建立FTP连接并传输文件

2.5 FTP配置命令

命令 功能
ftp server enable 使能FTP
set default ftp-directory flash:/ 设置FTP默认工作路径
aaa 进入AAA视图
local-user visiter password cipher visiter 配置用户名为visiter，密码为密文visiter
local-user visiter service-type ftp 配置用户接入类型为FTP
local-user visiter ftp-directory flash:/ 配置用户可访问的路径
local-user visiter access-limit 200 配置用户最大连接数为200
local-user visiter idle-timeout 0 0 配置用户闲置的超时时间
local-user visiter privilege level 3 配置用户权限为管理级别
ftp 1.1.1.1 访问FTP服务器地址1.1.1.1
binary 更改传输模式为二进制
get vrp.cc 传输文件 vrp.cc

Telnet用于远程连接、配置一台或多台设备，对这些网络设备进行集中的管理和维护

3 Telnet

3.1 Telnet优势

  • 提供交互式操作界面，允许终端远程登陆到任意一台可充当Telnet服务器的设备
  • 用户可以像本地登陆一样对远程设备进行操作
  • Telnet无需直连，只需保证两者之间的通信即可
  • 便于实现对设备的远程管理和维护

3.2 Telnet连接方式

  • Telnet 以 C/S模式运行

  • Telnet Server 和 Client 均是基于 TCP 连接来传输命令

  • Server 端口号默认为 23 号

    • Telnet 配置

  • password模式

命令 功能

user-interface maximum-vty 用于调整VTY接口的数量
