IPv6 地址分类

  • IPv6地址识别的是接口，而不是节点
  • 若要识别节点，需指定任意单播地址到此节点的任意一个接口
  1. 单播
    • 单薄地址识别此类型地址范围内的单个接口，地址范围指 IPv6 区域，区域内地址唯一
    • 使用恰当的单播技术，指向某单播地址的包能发送到单一的接口
    • 为实现负载均衡系统，RFC 4291 规定，多个接口对主机上的 IPv6 应用呈现为单一接口，允许该多接口使用同样的地址
  2. 组播
    • 多播识别相同或不同主机上的零个或多个接口
    • 使用恰当的多播技术，只想多播地址的包能发送到地址所标识的所有接口
  3. 任播
    • 泛播地址识别多个接口
    • 使用恰当的泛播技术，指向泛播地址的包会发送到地址所指定接口中最近的一个
    • 最近的接口指距离计算最近的接口

全球单播地址

相当于 IPv4 的公共地址，在 IPv6 网络上，全球可达、全球路由。

  • 最高位固定设置为 001
  • 全球路由前缀固定长度为 48 位，为公共拓扑，由各级 ISP 申请分配
  • 子网 ID 长度固定为 16 位，为站点拓扑，由机构站点分配子网
  • 接口 ID 长度固定为 64 位，为接口标识符，由子网上的唯一接口标识

本地使用的单播地址

  1. 链路本地地址
  2. 站点本地地址
  3. 唯一本地地址

链路本地地址

  • 固定前缀为：FE80::/64
  • 用于节点与同一链路上的邻节点通信
  • 对于某些 ND 过程是必须的

站点本地地址

  • 固定前10位为：FEC0::/10
  • 子网 ID 固定长度为 54 位，用于机构创建子网
  • RFC 3879 正式指出，不推荐在未来的 IPv6 实现中使用站点本地地址

不同于全球地址，链路本地地址和站点本地地址是可以重用的。链路本地地址在每个链路上重用，站点本地地址在机构的每个站点中重用，但是由于这种地址的重用，使得这两种地址具有歧义，为了明确指定目标所在链路、目的所在的站点，需要额外的区域标识符，即范围 ID，用于标识网络中由明确范围并连接的部分。

唯一本地地址

  • 固定前7位：1111 110
  • 第8位为本地 L 标签，为 1 时表示前缀实在本地分配的，为 0 时未配置
  • 故机构中的唯一本地地址前缀一般为 FD00::/8

特殊 IPv6 地址

  • 未指定路由
  • 回路地址

过渡地址

  • IPv4 映射地址
  • 6 to 4 地址
  • ISATAP 地址
  • Teredo 地址
