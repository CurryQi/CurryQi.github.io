1 传输介质

1.1 通信设备与介质

  • 通信设备：PC、服务器等
  • 通行传输介质：同轴电缆、双绞线、光纤等
  • 通信传输介质影响：线路编码方式、传输速度、传输距离

1.2 简单网络简介

  • 需求：终端互相传递信息、资源共享
  • 网络：一条承载数据传输的物理介质，连接两个终端，组成一个最简单的网络
    • 终端：产生数据、发送数据、接受数据
    • 介质：用来传输数据的物理载体
    • 网络：连接终端，建立通信的媒介；使用介质，进行数据传输
  • 性能指标：稳、快、准

1.3 通信介质

1.3.1 同轴电缆

名称 以太网标准 直径 最长有效传输距离 接头
粗缆 10BASE5 9.5mm 500米 N型接头
细缆 10BASE2 5mm 185米 BNC接头
  • 10BASE5 和 10BASE2 为两种早期以太网标准，均支持 10Mbps 的传输速率
  • 目前 10Mbps 无法满足企业网络需求，基本不用

1.3.2 双绞线(Twisted Pair)

以太网 线缆类别 最长有效传输距离
10BASE-T 两对3/4/5类双绞线 100米
100BASE-TX 两对5类双绞线 100米
1000BASE-T 四对5e类双绞线 100米
  • 屏蔽双绞线(Shielded Twisted Pair，STP)：双绞线与外层绝缘封套间，使用金属屏蔽层，屏蔽电磁干扰
  • 非屏蔽双绞线(Unshielded Twisted Pair, UTP)：无金属屏蔽层
    • 3 类双绞线支持 10Mbps 传输速率
    • 5 类双绞线支持 100Mbps 传输速率
    • 超 5 类 或更高级双绞线支持 千兆 传输速率
    • 双绞线使用 RJ-45 接头 连接网络设备，且接头针脚须按固定线序排列

1.3.3 光纤

以太网标准 线缆类别 最长有效传输距离
10BASE-F 单模/多模光纤 2000米
100BASE-FX 单模/多模光纤 2000米
1000BASE-LX 单模/多模光纤 316米
10000BASE-SX 多模光纤 316米
  • 单模光纤：只传输一种模式的光，无色散损耗，适合长距离高速传输
  • 多模光纤：传输多种模式的光，色散较大，信号脉冲展宽严重，只能短距离传输

1.3.4 串口电缆

线缆类别 速率 串口电缆标准 传输距离
V.24 1.2Kbit/s~64Kbit/s RS-232(DB-9、DB-25) 6米
V.35 1.2Kbit/s~2.048Mbit/s RS-422、RS-484 1200米
  • 目前 RS-232 逐渐被 FireWire 、 USB 等新标准取代
  • 新产品和设备普遍使用 USB 标准

1.4 冲突域、冲突检测

  • 多个主机使用同一根同轴电缆连接通信，该同轴电缆被称为共享介质，此网络称为共享介质网络。
  • 当网络中不同主机同时发送数据时，即产生信号冲突的问题
  • 一般是采用 载波侦听多路访问 / 冲突检测技术 (CSMA/CD)
    • 终端设备不停检测共享线路状态：若线路空闲，即发送数据；若线路繁忙，延时发送数据（延时时间由退避算法决定）
      1. 若存在两个或两个以上设备同时发送数据，线路发生数据冲突
      2. 终端设备检查到冲突信号后，马上停止发送自己的数据，并发送阻塞信息，强化冲突信号，使其他站点尽快检测冲突
      3. 终端设备检测到冲突信号后，延时等待一段时间后再进行数据发送（延时时间由退避算法决定）
  • CSMA/CD 的工作原理总结：先听先发，边听边发，冲突停发，随机延迟后重发

1.5 通信模式

  • 设备工作模式
    • 单工模式：设备只能收，或只能发
    • 半双工模式：设备能收，也能发，但同一时间只能收，或只能发
    • 全双工模式：设备能同时进行收、发
  • 以太网工作模式
    • 半双工模式：冲突域内设备必须采用 CSMA/CD 机制避免冲突
    • 全双工模式：该模式不会产生冲突
  • 同一物理链上相连的两台设备双工模式必须一致

2 以太网帧

网络传输数据需要遵循标准，以太网根据 IEEE 802.3 标准管理和控制数据帧

2.1 网络通信协议种类

OSI TCP/IP IPX/SPX SNA
局域网 广域网 广域网 广域网
IEEE 802 以太网 PPP HDLC
  • 协议和标准两个词汇常常可以混用，且具有层次的特点
  • 通常，关注于逻辑数据关系的协议称为 上层协议 ；关注于物理数据流的协议称为 底层协议
    • IEEE 802 为一套管理物理数据流在局域网中传输的标准，包含在局域网中传输物理数据的 802.3 以太网标准
    • PPP(Point-to-Point Protocol) 为一套管理物理数据流在 广域网 的传输标准
    • HDLC(High-Level Data Link Control) 为一套管理物理数据流在 广域网 的传输标准

2.2 OSI七层模型

名称 作用
应用层 为应用程序提供网络服务
表示层 数据格式化、加密、揭秘：提供各种用于应用层数据的编码和转换功能，确保系应用层发送的数据能够被另外的系统应用层识别
会话层 建立、维护、管理 会话连接 ：该层的通信由不同设备中应用程序之间的服务请求和响应组成
传输层 建立、维护、管理 端到端连接 ：提供面向连接(TCP)、非面向连接的数据传输(UDP)，并进行重传前的差错检测
网络层 IP寻址和路由选择 ：提供逻辑地址，供路由器确定传输路径
数据链路层 控制网络层和物理层之间的通信：将比特组合成字节，将字节组合成帧，使用链路层地址访问介质，进行差错控制
物理层 设备之间传输比特流：规定电平、速度、电缆针脚
  • OSI分层模型优点：
    • 简化相关网络操作
    • 提供不同厂商兼容性
    • 促进标准化工作
    • 结构上分层，易于学习和操作

2.3 TCP/IP分层模型

层名称 数据集合名称 英文及简写
应用层 数据 PDU
传输层 数据段 Segment
网络层 数据包 Packet
网络结构层 数据帧、比特流 Frame、Bit
  • TCP/IP模型继承了OSI模型的分层结构，层与层相对独立，且具有密切的协作关系
  • TCP/IP分四层模型不关注底层物理介质，主要关注终端间的逻辑转发问题，故TCP/IP核心为网络层、传输层：
    • 网络层：解决网络之间的逻辑转发问题
    • 传输层：保证源端到目的端之间的可靠传输
  • 数据 $\xrightarrow{TCP报头}$ 数据段 $\xrightarrow{IP报头}$ 数据包 $\xrightarrow{数据链路层报头}$ 数据帧 $\rightarrow$ 比特流

2.4 以太网常见数据帧格式

2.4.1 Ethernet II（以太网多用）

D.MAC S.MAC Type Data FC5
目的地址 源地址 类型 数据 循环冗余校验
6 Byte 6 Byte 2 Byte 46-1500 Byte 4 Byte

2.4.2 IEEE 802.3

D.MAC S.MAC Length LLC SNAP Data FC5
目的地址 源地址 长度 逻辑链路控制 子网接入协议 数据 循环冗余校验
6 Byte 6 Byte 2 Byte 3 Byte 5 Byte 38-1492 Byte 4 Byte

2.4.3 LLC与SNAP

LLC:3 Byte SNAP:5 Byte
D.SAP S.SAP Control Org Code Type
目的服务访问点 源服务访问点 控制 机构代码 类型
3 Byte 1 Byte 1 Byte 3 Byte 2 Byte
  • 若 Length/Type >= 1546(0x0600)，为 Ethernet II；若 Length/Type <= 1500(0x05DC)，为 IEEE802.3
  • 循环冗余校验（FCS）提供错误检测机制，用于检验传输过程帧的完整性
  • 两种Type字段均用于标识数据字段高层协议：若Type=0x0800，则为IP协议帧；若Type=0x0806，则为ARP协议帧
  • Org Code三个字节均为 0
  • 当 D.SAP 和 S.SAP 均取特定值 0xff时，802.3帧变为Netware-ETHERNET帧，用于承载NetWare类型数据
  • 当 D.SAP 和 S.SAP 均取特定值 0xaa 时，802.3帧变为ETHERNET_SNAP帧，用于传输多种协议
  • 当 D.SAP 和 S.SAP 其他取值时均为纯 IEEE 802.3帧

2.5 以太网的MAC地址

  • 数据链路层基于MAC地址进行帧的传输，以太网在二层链路上通过MAC地址来唯一标示网络设备，并实现局域网上的通信
  • MAC地址也叫做物理地址，大多数厂商直接将MAC地址烧入网卡的ROM中，通信发送端使用接收端的MAC地址作为目的地址
  • MAC地址由两部分组成：
48 bits
24 bits 24 bits
OUI(组织唯一标示符号) 由供应商分配
  • 网络设备的MAC地址是全球唯一的，长度为48比特，通常用16进制表示
  • 组织唯一标示符由IEEE统一分配给设备制造商，华为网络产品的MAC地址前24比特为0x00e0fc，后24位由厂商自行分配
  • 帧的发送方式：单播、广播、组播
